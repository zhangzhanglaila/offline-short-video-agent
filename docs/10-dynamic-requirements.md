# 📋 动态化视频 需求规范

## 问题定义

**用户反馈（2026-07）**:
> "就是再放一张一张的ppt，没有任何交互，页面里面没有任何元素的交互，
> 照片不能平铺整个画面。发散思维让视频更加多样化、吸睛。"

### 根因分析
当前合成管线：**每个场景渲染1张静态PNG → 定格N秒 → 拼接**。
导致三大问题：

| 问题 | 现状 | 根因 |
|------|------|------|
| 静态如PPT | 画面完全静止 | 单张图定格，无帧间运动 |
| 无元素交互 | 文字/素材一次性全显示 | 无入场动画、无编排 |
| 照片不"满" | 底部30%被字幕条压暗 | 字幕条过重 + 无运动感 |

---

## 目标

把"静态幻灯片"升级为"有运动、有编排、有层次"的动态短视频。

### 核心目标
1. **画面有运动**: 背景素材缓慢运镜（推拉/平移），不再死板静止
2. **元素有入场**: 文字、要点、图标逐个动态出现，而非一次铺满
3. **画面更通透**: 素材全屏铺满(full-bleed) + 更轻盈的字幕处理
4. **切换更多样**: 场景转场丰富化（不止交叉淡入）
5. **背景可动**: 支持真实视频素材作为动态背景

### 衡量标准
| 指标 | 当前 | 目标 |
|------|------|------|
| 画面运动 | 0（静止） | 每个内容场景有运镜 |
| 元素入场动画种类 | 0 | ≥3种（淡入/滑入/逐字） |
| 素材可视占比 | ~70%（字幕遮挡） | ≥90% full-bleed |
| 转场种类 | 1（xfade） | ≥4种 |
| 背景类型 | 仅静图 | 静图 + 视频素材 |

---

## 功能需求

### FR1: 运动镜头 (Ken Burns)
- 背景静图应用缓慢缩放+平移
- 运镜方向/幅度可按场景变化（避免每个场景雷同）
- 素材真正全屏铺满，无黑边

### FR2: 元素入场动画
- 标题：逐字打字 / 逐行上滑 / 淡入放大
- 字幕：淡入 / 从底部滑入
- 关键词：高亮 / 放大强调
- 多个元素按时间轴**先后出现**（制造"编排感"）

### FR3: 场景内分层与元素
- 场景 = 多个图层（背景 / 字幕 / 装饰元素 / 强调标注）
- 支持图标、形状、要点列表、序号
- 元素间有出现顺序（用户要的"交互感"）

### FR4: 丰富转场
- 复用已有 transition_effects.py 的13种
- 按内容类型/节奏智能选择转场

### FR5: 视频素材背景
- 支持 Pexels/Pixabay 视频素材（复用 stock_video_module.py）
- 视频与静图混合使用
- 视频过短时循环/放慢

### FR6: 节奏与音乐（可选增强）
- 背景音乐 + 节拍同步剪辑（复用 rhythm_engine + audio_analyzer）
- TTS旁白（复用 tts_module，narration字段已预留）

---

## 非功能需求

| 项 | 要求 |
|----|------|
| 性能 | 30s视频合成 < 3分钟（运动增加渲染量，需控制） |
| 容错 | 每种动效失败可降级为静态（保持"永远有输出"原则） |
| 兼容 | 复用现有Agent架构，不破坏已完成的Phase 0-5 |
| 可测 | 每个动效模块可独立单元测试 |
| 渐进 | 分阶段交付，每阶段可见效果提升 |

---

## 约束与原则

1. **不推倒重来**: 在现有VideoComposeAgent基础上增强，不重写架构
2. **复用优先**: 优先接入已有的动效资产（text_animator等7个模块）
3. **降级保底**: 任何动效失败都回退到当前的静态渲染
4. **稳健推进**: 一次一个动效层，充分测试再进下一步
5. **密钥安全**: 延续.env管理，不入git

## 范围边界

### 本次包含
- 运镜、元素动画、分层、转场、视频素材

### 本次不包含（后续）
- 3D效果、AI生成视频、复杂粒子系统
- 实时预览编辑器
- 多语言字幕
